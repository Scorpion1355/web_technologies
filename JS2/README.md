# Part 1 — Refactor (part_1.js)

У файлі вже є реалізація 3 функцій.
Ваша задача — порефакторити код, зберігши поведінку відповідно до тестів, а також доповнити логіку, щоб проходили додаткові тести.

---

## 1. max(arr)

Повертає максимальне число з масиву.

Контракт:

- Якщо arr не масив або масив порожній → повернути NaN
- Вхідний масив не має мутуватися

---

## 2. maxPair(arr)

Повертає пару сусідніх значень [a, b],
для яких сума a + b максимальна.

Правила:

- Пара формується тільки із сусідніх елементів
- Для останнього елемента, якщо пари немає — вважати b = 0
- Якщо масив порожній або не масив → NaN
- Масив не має мутуватися

Приклади:
[1,2,3,4,5] → [4,5]
[10] → [10,0]

---

## 3. recordProgress(obj, tuple)

Повертає новий об’єкт, в якому:

- age збільшено на tuple[0]
- weight збільшено на tuple[1]

Контракт:

- obj має бути звичайним об’єктом (не null, не масив)
- інакше → throw new Error("Invalid object")
- якщо tuple порожній → повернути копію obj
- якщо obj не має age або weight → вважати їх рівними 0
- оригінальний об’єкт не має мутуватися

# Part 2A — Prototype version (part_2.prototype.js)

Реалізуйте систему фігур з використанням prototype-based inheritance.

Потрібні конструктори:

- Shape(type)
- Square(size)
- Rectangle(width, height)
- Circle(radius)

---

## Вимоги

1. Метод getArea()

У кожної фігури має бути метод getArea():

Square(10) → 100
Rectangle(5, 10) → 50
Circle(2) → Math.PI \* 4

---

2. Перетворення в рядок

String(shape)
`${shape}`

має повертати:

[Shape rectangle]
[Shape square]
[Shape circle]

---

3. Арифметика

Об’єкти повинні коректно працювати в арифметичних виразах:

rectangle + square - circle

Результат має бути числом (на основі площі).

---

4. Наслідування

instanceof має працювати коректно.
